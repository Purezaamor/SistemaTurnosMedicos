# Anexo - Principios SOLID

Los Principios SOLID son un conjunto de buenas prácticas dentro del Diseño Orientado a Objetos que existen para evitar que el código se vuelva difícil de entender, mantener o escalar con el tiempo. Básicamente, nacen porque cuando un sistema crece sin una estructura clara, termina lleno de clases desordenadas, muy acopladas y complicadas de modificar sin romper otras cosas. SOLID propone reglas simples para organizar mejor el código: dividir responsabilidades, hacer que las clases sean más independientes y permitir que el sistema evolucione sin tener que rehacer todo desde cero.
