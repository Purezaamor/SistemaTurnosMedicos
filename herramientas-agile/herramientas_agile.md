## Descripción

Las tarjetas CRC (Class-Responsibility-Collaboration) permiten modelar el comportamiento del sistema identificando responsabilidades y colaboraciones entre las distintas clases.

Este artefacto se utilizó para validar el diseño orientado a objetos del sistema de turnos médicos, asegurando una correcta asignación de responsabilidades y una adecuada interacción entre las entidades del dominio.

## Actividad realizada

En esta actividad se llevó a cabo:

- Identificación de las clases principales del sistema
- Definición de responsabilidades de cada clase
- Determinación de sus colaboraciones
- Modelado del pensamiento del objeto
- Especificación de propiedades relevantes
- Incorporación de una clase de auditoría (**HistorialCambios**) para garantizar la trazabilidad del sistema

Las tarjetas fueron organizadas en archivos individuales dentro de la carpeta `tarjetas-crc/`, siguiendo la estructura solicitada en la consigna.
